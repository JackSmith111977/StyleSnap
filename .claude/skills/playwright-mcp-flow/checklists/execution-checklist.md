# Playwright MCP 执行检查清单

## 执行前检查（Pre-Flight）

### 环境状态
- [ ] 目标服务器已启动（如 `pnpm dev`）
- [ ] 无残留浏览器进程
- [ ] Playwright MCP 服务可用
- [ ] 测试 URL 可访问

### 浏览器状态
- [ ] 当前无已打开的浏览器窗口
- [ ] 如存在残留，已执行 `browser_close`
- [ ] 已记录初始状态（截图/快照）

---

## 启动阶段检查

### 浏览器启动
- [ ] 调用 `browser_eval start` 或 `browser_start`
- [ ] 等待 3-5 秒确保浏览器完全启动
- [ ] 确认浏览器窗口 visible
- [ ] 记录启动时间

### 导航配置
- [ ] 目标 URL 正确
- [ ] 设置了合适的超时时间
- [ ] 准备好等待策略（如等待特定元素）

---

## 执行阶段检查

### 导航验证
- [ ] 调用 `browser_navigate` 导航到目标页
- [ ] 页面加载完成（检查 loading 状态消失）
- [ ] 验证页面标题或关键元素存在
- [ ] 检查控制台是否有启动错误

### 交互操作
- [ ] 使用正确的 ref/selector 定位元素
- [ ] 操作前等待元素可交互（enabled/visible）
- [ ] 每次操作后验证预期结果
- [ ] 记录每个操作步骤和时间

### 验证检查
- [ ] 页面 URL 符合预期
- [ ] 关键元素存在且可见
- [ ] 预期文本/内容显示
- [ ] 无控制台错误（或错误可解释）
- [ ] 网络请求状态正确（如适用）

---

## 执行后检查（Post-Flight）

### 证据保存
- [ ] 关键状态已截图
- [ ] 控制台日志已保存
- [ ] 网络请求已记录（如适用）
- [ ] 测试记录已填写

### 清理流程（重要！）
- [ ] 已调用 `browser_close` 关闭浏览器
- [ ] 确认浏览器窗口已关闭
- [ ] 检查无残留进程
- [ ] 在测试记录中标注「已清理」

### 状态确认
- [ ] 测试状态标记为「完成」
- [ ] 问题已记录（如有）
- [ ] 后续行动已明确
- [ ] 相关人员已通知（如需要）

---

## 快速检查命令

```bash
# 检查浏览器进程（Windows）
tasklist | findstr -i "chrome msedge"

# 关闭浏览器进程（如需手动清理）
taskkill /F /IM chrome.exe /IM msedge.exe

# Playwright MCP 命令
browser_close          # 正常关闭
browser_eval start     # 启动浏览器
browser_snapshot       # 获取页面快照
browser_console_messages level="error"  # 检查错误
```

## 常见失败场景

| 场景 | 检查项 | 解决方案 |
|------|--------|----------|
| 浏览器启动失败 | MCP 服务状态 | 重启 Playwright MCP |
| 导航超时 | URL 是否正确、服务器是否启动 | 检查服务器状态、增加超时 |
| 元素找不到 | selector 是否正确 | 使用 browser_snapshot 检查当前 DOM |
| 操作无效 | 元素是否可交互 | 等待元素 enabled/visible |
| 关闭失败 | 浏览器是否卡住 | 等待 5 秒后重试，或手动清理进程 |
