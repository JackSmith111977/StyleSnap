# 代码审查指南

## 用途

当用户请求代码审查时，自动使用此 Skill。

**适用场景：**
- Pull Request 审查
- 代码提交前检查
- 安全审计

---

## 审查范围

### 审查类型

- [ ] 安全检查
- [ ] 性能分析
- [ ] 代码质量
- [ ] 测试覆盖率

### 审查语言

- [支持的语言 1]
- [支持的语言 2]

---

## 检查清单

### 安全检查

| 检查项 | 说明 | 严重级别 |
|--------|------|----------|
| SQL 注入 | 检查字符串拼接查询 | 🔴 高危 |
| XSS 攻击 | 检查用户输入转义 | 🔴 高危 |
| 认证授权 | 检查权限验证 | 🔴 高危 |
| 敏感数据 | 检查加密存储 | 🟡 中危 |

---

### 性能检查

| 检查项 | 说明 | 严重级别 |
|--------|------|----------|
| N+1 查询 | 检查循环查询数据库 | 🟡 中危 |
| 内存泄漏 | 检查资源释放 | 🟡 中危 |
| 大列表 | 检查分页处理 | 🟢 低危 |

---

### 代码质量检查

| 检查项 | 说明 |
|--------|------|
| 命名规范 | 变量/函数命名清晰 |
| 函数长度 | 单函数不超过 50 行 |
| 注释质量 | 关键逻辑有注释说明 |
| 重复代码 | 无明显重复代码 |

---

## 审查流程

### 步骤 1：阅读变更

- 读取变更文件
- 理解变更目的
- 识别影响范围

### 步骤 2：执行检查

按检查清单逐项检查。

### 步骤 3：生成报告

按以下格式输出：

```markdown
## 审查结果

### 🔴 高危问题（X 个）

1. [问题描述]
   - 位置：`文件：行号`
   - 风险：[风险说明]
   - 修复：[修复建议]

### 🟡 中危问题（X 个）

...

### 🟢 建议改进（X 个）

...
```

---

## 踩坑清单

### 常见陷阱 1：SQL 注入

❌ **错误：**
```python
query = f"SELECT * FROM users WHERE id = {user_id}"
```

✅ **正确：**
```python
query = "SELECT * FROM users WHERE id = ?"
cursor.execute(query, [user_id])
```
