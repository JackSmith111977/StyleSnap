# Skill 创建检查清单

> 在 Skill 生成后/写入前必须执行此检查

---

## 一、YAML Frontmatter 检查

### 必需字段

- [ ] `name` 字段存在且格式正确（小写字母 + 数字 + 连字符）
- [ ] `name` 长度在 1-64 字符内
- [ ] `description` 字段存在且清晰具体
- [ ] `description` 长度在 1-1024 字符内
- [ ] `description` 避免笼统描述（如"处理代码相关任务"）

### 可选字段

- [ ] `aliases` 格式正确（数组）
- [ ] `triggers` 格式正确（数组，3-5 个触发词）
- [ ] `author` 字段已填写
- [ ] `version` 字段已填写（建议从 1.0.0 开始）

---

## 二、内容结构检查

### 必需章节

- [ ] 包含"用途"章节（适用场景）
- [ ] 包含"工作流程"章节（具体步骤）
- [ ] 包含"输出格式"章节（规范定义）
- [ ] 包含"注意事项"章节（边界情况）

### 推荐章节

- [ ] 包含"检查清单"章节
- [ ] 包含"踩坑清单"章节
- [ ] 包含"示例"章节
- [ ] 包含"参考资源"章节

---

## 三、内容深度检查

### 用例定义

- [ ] 明确定义了 2-3 个具体用例
- [ ] 每个用例包含触发条件、执行步骤、预期结果

### 工作流程

- [ ] 工作流程步骤清晰
- [ ] 每个步骤有具体操作说明
- [ ] 包含使用的工具/命令

### 输出格式

- [ ] 定义了输出结构
- [ ] 包含格式示例
- [ ] 有明确的规范要求

---

## 四、最佳实践检查

### 避免事项

- [ ] 没有陈述显而易见的内容
- [ ] 没有模糊的指令（如"根据需要处理"）
- [ ] 没有遗漏关键步骤

### 推荐事项

- [ ] 包含可执行的检查清单
- [ ] 包含踩坑清单（Gotchas）
- [ ] 包含具体示例（好坏对比）
- [ ] 定义了清晰的输出格式

---

## 五、触发配置检查

### 显式触发

- [ ] 配置了 `/skill-name` 命令
- [ ] 命令名称与 `name` 字段一致

### 隐式触发

- [ ] 配置了 3-5 个中文触发词
- [ ] 触发词覆盖常见表达方式
- [ ] 触发词与 Skill 用途相关

---

### 目录结构

- [ ] Skill 目录已创建
- [ ] `SKILL.md` 文件存在
- [ ] 目录名与 `name` 字段一致

### 可选文件

- [ ] `templates/` 目录（如有需要）
- [ ] `references/` 目录（如有需要）
- [ ] `scripts/` 目录（如有需要）
- [ ] `checklists/` 目录（如有需要）

- [ ] 表格有表头分隔线
- [ ] 列表格式正确

### YAML 格式

- [ ] Frontmatter 以 `---` 开始和结束
- [ ] 数组格式正确
- [ ] 字符串引号正确
