# Skill 修改检查清单

> 在修改已有 Skill 时执行此检查

---

## 一、修改前准备

### 备份检查

- [ ] 已读取原有 Skill 内容
- [ ] 已识别需要修改的章节
- [ ] 已确认修改范围

### 影响分析

- [ ] 分析修改对现有功能的影响
- [ ] 分析修改对触发机制的影响
- [ ] 分析修改对其他 Skill 的影响

---

## 二、修改内容检查

### 新增内容

- [ ] 新增内容与原有内容风格一致
- [ ] 新增内容没有重复已有内容
- [ ] 新增内容有清晰的章节归属

### 修改内容

- [ ] 修改内容准确反映用户需求
- [ ] 修改内容与上下文兼容
- [ ] 修改后逻辑自洽

### 删除内容

- [ ] 确认删除内容不再需要
- [ ] 确认删除不影响核心功能
- [ ] 确认没有破坏引用关系

---

## 三、版本管理

### 更新记录

- [ ] 在文档末尾添加"更新记录"章节
- [ ] 记录版本号、日期、修改内容

### 版本规范

- [ ] 主版本号：不兼容的修改
- [ ] 次版本号：向后兼容的新功能
- [ ] 修订号：向后兼容的问题修复

---

## 四、修改后检查

### 完整性检查

- [ ] 修改后 Skill 仍然完整
- [ ] 修改后工作流程仍然连贯
- [ ] 修改后检查清单仍然有效

### 一致性检查

- [ ] 修改后内容风格一致
- [ ] 修改后术语使用一致
- [ ] 修改后格式规范一致

---

## 五、用户确认

### 修改摘要

- [ ] 输出修改摘要供用户确认
- [ ] 标注新增内容
- [ ] 标注修改内容
- [ ] 标注删除内容

### 用户反馈

- [ ] 等待用户确认修改
- [ ] 根据用户反馈调整
- [ ] 用户确认后再写入

---
