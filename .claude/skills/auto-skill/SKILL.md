# auto-skill - 重复流程检测与 Skill 建议

## 用途

当用户反复执行相似类型的命令时，自动检测并在达到阈值后询问是否创建专用 Skill。

**核心原则：**
1. 后台计数：静默追踪用户命令模式，不打扰用户
2. 阈值触发：达到配置次数（默认 3 次）后才询问
3. 手动询问：由 Claude 主动提问，用户决定是否创建
4. 关键词匹配：相似意图的命令归为同一模式

---

## 工作流程

### 阶段 1：后台计数（自动执行）

**步骤 1.1：Hook 自动触发**

每次用户提交提示词时，`UserPromptSubmit` Hook 自动执行：

```bash
node .claude/scripts/count-tracker.js --track "$USER_PROMPT"
```

**步骤 1.2：关键词匹配**

脚本从用户输入中提取匹配的关键词：

| 用户输入 | 匹配模式 |
|----------|----------|
| "帮我部署项目" | deploy |
| "运行测试" | test |
| "审查这个 PR" | code-review |
| "创建 Docker 镜像" | docker |

**步骤 1.3：更新计数文件**

读写 `.claude/command-counts.json`：

```json
{
  "threshold": 3,
  "patterns": {
    "deploy": {
      "count": 2,
      "lastUsed": "2026-03-25T10:30:00Z",
      "examples": ["帮我部署项目", "部署到生产环境"],
      "triggered": false
    }
  }
}
```

---

### 阶段 2：阈值检测与标记输出

**步骤 2.1：检测触发条件**

当某个模式的计数达到阈值时，脚本输出特殊标记：

```
[AUTO-SKILL-TRIGGER] 检测到重复命令模式！
模式：deploy
次数：3 次
示例：帮我部署项目 | 部署到生产环境

建议：是否要为 "deploy" 类操作创建一个专用 Skill？
```

**步骤 2.2：Claude 检测到标记**

当输出包含 `[AUTO-SKILL-TRIGGER]` 标记时，Claude 主动询问用户。

---

### 阶段 3：用户询问与 Skill 创建

**步骤 3.1：Claude 主动询问**

```markdown
📊 **重复模式检测**

我注意到您最近多次执行类似的操作：

**模式：** deploy（部署类）
**次数：** 3 次
**示例命令：**
- 帮我部署项目
- 部署到生产环境

是否为这类操作创建一个专用 Skill？创建后您可以通过 `/deploy` 命令快速触发。

**预计 Skill 功能：**
- 自动化构建流程
- 环境配置检查
- 部署状态报告

请回复「是」或「否」。
```

**步骤 3.2：用户确认后创建**

用户确认后，调用 `skill-creator` Skill 协助创建：

```
/skill-creator 创建 deploy Skill，用于自动化部署流程
```

**步骤 3.3：重置计数**

创建完成后，重置该模式的计数：

```bash
node .claude/scripts/count-tracker.js --reset deploy
```

---

### 阶段 4：手动触发（备用方式）

**步骤 4.1：用户主动查询**

用户可随时使用 `/auto-skill` 命令查看当前状态：

```
/auto-skill 查看当前追踪的命令模式
```

**步骤 4.2：输出状态报告**

```markdown
## 当前追踪的命令模式

| 模式 | 计数 | 状态 |
|------|------|------|
| deploy | 2/3 | 📊 正常 |
| test | 1/3 | 📊 正常 |
| docker | 3/3 | ⚠️ 已触发 |

**已触发模式：** docker
建议：是否为 Docker 相关操作创建专用 Skill？
```

## 踩坑清单

| 陷阱 | 错误做法 | 正确做法 |
|------|----------|----------|
| **过度打扰** | 每次都询问 | 仅在达到阈值后询问一次 |
| **匹配过宽** | 单关键词匹配 | 多关键词组合，提高准确性 |
| **计数污染** | 不重置计数 | 创建 Skill 后重置计数 |
| **忽略用户意愿** | 强制创建 | 始终询问，用户决定 |
| **文件权限** | 不检查文件存在 | 读取前检查文件是否存在 |

## 注意事项

### 性能考虑
- 脚本执行时间 < 100ms，不影响用户体验
- 计数文件采用原子写入，避免并发冲突

### 隐私保护
- 仅记录命令模式类别，不记录完整命令内容
- 每个模式最多保存 3 个示例

### 限制说明
- 无法检测 Shell 命令的历史输出
- 依赖用户主动输入，被动执行无法追踪

## 配置

### 阈值配置

编辑 `.claude/command-counts.json`：

```json
{
  "threshold": 3  // 修改此值调整触发次数
}
```

### 模式配置

编辑 `.claude/scripts/count-tracker.js` 中的 `COMMAND_PATTERNS` 对象，添加或删除关键词模式。

### Hook 配置

编辑 `.claude/settings.local.json`：

```json
{
  "hooks": {
    "UserPromptSubmit": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "node .claude/scripts/count-tracker.js --track \"$USER_PROMPT\"",
            "timeout": 5
          }
        ]
      }
    ]
  }
}
```
