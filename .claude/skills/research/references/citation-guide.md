# 引用格式规范

> 所有调研内容必须标注来源，确保透明可追溯

---

## 来源优先级

| 优先级 | 来源类型 | 可信度 | 使用要求 |
|--------|----------|--------|----------|
| 1 | 官方文档 | ⭐⭐⭐⭐⭐ | 冲突时以此为准 |
| 2 | 技术博客/社区文章 | ⭐⭐⭐ | 需交叉验证 |
| 3 | 源码 | ⭐⭐⭐⭐ | 标注文件路径和行号 |
| 4 | 视频内容 | ⭐⭐ | 记录时间戳 |
| 5 | 学术论文 | ⭐⭐⭐⭐ | 完整引用格式 |

---

## 文中标注格式

### 单一来源

```markdown
这是一个技术描述。[官方文档 #1]
```

### 多个来源

```markdown
另一个描述，存在不同观点。[技术博客 #2] [官方文档 #1]
```

### 冲突观点

```markdown
关于这个问题有两种观点：
- 观点 A [官方文档 #1]
- 观点 B [技术博客 #2]

**建议采用：** 观点 A（官方文档）
```

## 各类型引用格式

### 官方文档

| 字段 | 格式 |
|------|------|
| 来源类型 | 官方文档 |
| 标题 | [项目名] 官方文档 - [章节名] |
| 链接 | 完整 URL |
| 查阅时间 | YYYY-MM-DD |

### 技术博客

| 字段 | 格式 |
|------|------|
| 来源类型 | 技术博客 |
| 标题 | [作者] - [文章标题] |
| 链接 | 完整 URL |
| 查阅时间 | YYYY-MM-DD |
| 备注 | 可选：作者背景/文章质量 |

### 源码

| 字段 | 格式 |
|------|------|
| 来源类型 | 源码 |
| 标题 | [仓库名] - [文件路径] |
| 链接 | GitHub/GitLab URL |
| 查阅时间 | YYYY-MM-DD |
| 备注 | 标注关键行号 |

### 视频内容

| 字段 | 格式 |
|------|------|
| 来源类型 | 视频内容 |
| 标题 | [UP 主] - [视频标题] |
| 链接 | 完整 URL |
| 查阅时间 | YYYY-MM-DD |
| 备注 | 标注平台和时间戳 |

### 学术论文

| 字段 | 格式 |
|------|------|
| 来源类型 | 学术论文 |
| 标题 | [完整论文标题] |
| 链接 | DOI 或期刊链接 |
| 查阅时间 | YYYY-MM-DD |
| 备注 | 期刊/会议名称 |

## 信息冲突处理

当不同来源存在冲突时：

```markdown
## 冲突处理

**问题：** [描述冲突点]

| 来源 | 观点 | 可信度 |
|------|------|--------|
| 官方文档 | [观点 A] | ⭐⭐⭐⭐⭐ |
| 技术博客 A | [观点 B] | ⭐⭐⭐ |
| 技术博客 B | [观点 C] | ⭐⭐⭐ |

**建议采用：** 观点 A（官方文档）

**说明：** [解释为什么采用该观点]
```
