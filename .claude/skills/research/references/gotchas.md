# 踩坑清单（Gotchas）

> 调研整理过程中的常见错误及避免方法

---

## 陷阱 1：来源单一

### ❌ 错误做法

只依赖单一来源（如只看官方文档或只看某篇博客）

### ✅ 正确做法

至少查阅 3-5 个不同来源，交叉验证信息

### 原因

单一来源可能存在：
- 偏见或立场问题
- 信息过时
- 内容不完整

### 检查方法

```markdown
## 来源检查
- [ ] 官方文档已查阅
- [ ] 至少 2 个技术博客已查阅
- [ ] 不同作者的观点已对比
```

---

## 陷阱 2：缺少引用

### ❌ 错误做法

直接复制内容而不标注来源

### ✅ 正确做法

所有引用内容标注来源，使用统一的引用格式

### 原因

- 避免抄袭嫌疑
- 方便读者查证
- 提高文档可信度

### 引用格式

```markdown
这是一个技术描述。[官方文档 #1]
```

---

## 陷阱 3：内容过浅

### ❌ 错误做法

只罗列 API 用法，不解释原理

### ✅ 正确做法

每个核心概念包含「定义 + 原理 + 示例 + 误区」

### 示例对比

**❌ 过浅：**
```markdown
### useState

```jsx
const [state, setState] = useState(initialValue)
```
```

**✅ 深度：**
```markdown
### useState

**定义：** 用于在函数组件中添加状态管理的 Hook

**工作原理：**
useState 内部维护一个状态链表，每个 Hook 按调用顺序存储在链表中...

**示例：**
```jsx
const [count, setCount] = useState(0);
```

**注意：** setCount 是异步的，多次调用会合并更新
```

## 陷阱 4：忽视冲突观点

### ❌ 错误做法

遇到不同观点时随意选择一个

### ✅ 正确做法

明确标注冲突观点，分析原因，给出建议

### 格式

```markdown
## 冲突观点

| 来源 | 观点 | 可信度 |
|------|------|--------|
| 官方文档 | [观点 A] | ⭐⭐⭐⭐⭐ |
| 技术博客 | [观点 B] | ⭐⭐⭐ |

**建议采用：** 观点 A（官方文档）
**说明：** [解释原因]
```

---

## 陷阱 5：不更新过时信息

### ❌ 错误做法

发现文档有旧内容直接覆盖

### ✅ 正确做法

检测已有文档，在基础上更新并添加更新记录

### 更新记录格式

```markdown
---

## 更新记录

| 版本 | 日期 | 更新内容 | 作者 |
|------|------|----------|------|
| 1.1.0 | 2026-03-25 | 新增 React 19 特性章节 | Kei |
| 1.0.0 | 2026-03-20 | 初始版本 | Kei |
```

---

## 陷阱 6：忽视时效性

### ❌ 错误做法

不标注信息的时效性，使用过时信息

### ✅ 正确做法

- 标注查阅时间
- 注明版本信息
- 定期更新文档

### 示例

```markdown
*文档创建日期：2026-03-25*
*最后更新：2026-03-25*
*版本：1.0.0*
*适用版本：React 18+*
```

---

## 陷阱 7：缺乏结构

### ❌ 错误做法

内容堆砌，没有清晰的章节结构

### ✅ 正确做法

使用标准 8 章节结构：
1. 概述
2. 核心概念
3. 快速入门
4. 基础用法
5. 高级特性
6. 实战案例
7. 常见问题/陷阱
8. 学习资源推荐

---

## 陷阱 8：代码示例不可运行

### ❌ 错误做法

给出片段代码，缺少上下文或依赖

### ✅ 正确做法

- 提供完整可运行的示例
- 包含必要的导入和依赖
- 添加注释说明

---

## 检查清单

在文档完成前检查：

```markdown
- [ ] 至少 3-5 个不同来源
- [ ] 所有引用标注来源
- [ ] 核心概念包含定义 + 原理 + 示例 + 误区
- [ ] 冲突观点已标注
- [ ] 更新记录已添加
- [ ] 查阅时间已记录
- [ ] 代码示例可运行
```
