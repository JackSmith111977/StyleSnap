# 阶段门控检查清单

**返回主文件：** [SKILL.md](../SKILL.md)

---

## 阶段 1 → 阶段 2：事件链完整性检查

- [ ] 问题描述清晰，能够定位到具体功能模块
- [ ] 已绘制从用户操作到后端处理的完整事件链
- [ ] 已识别所有关键节点（至少 3 个）
- [ ] 已标记可能出错的环节

**门控问题**:
1. 能否清晰描述从用户操作到数据处理的完整流程？
2. 是否识别了所有可能出错的节点？

---

## 阶段 2 → 阶段 3：调试输出覆盖检查

- [ ] 前端事件处理函数已添加日志
- [ ] Server Action 已添加日志
- [ ] 认证/授权函数已添加日志（如适用）
- [ ] 外部调用已添加日志
- [ ] 关键变量已打印

**门控问题**:
1. 每个关键节点是否都有日志输出？
2. 日志是否包含足够的上下文信息？

---

## 阶段 3 → 阶段 4：MCP 调试输出检查

- [ ] 已启动开发服务器
- [ ] 已使用 nextjs_index 发现服务器
- [ ] 已使用 nextjs_call get_errors 获取错误
- [ ] 已导航到问题页面
- [ ] 已执行问题操作
- [ ] 已获取控制台消息
- [ ] 已从日志中定位问题环节

**门控问题**:
1. 是否获取了运行时的完整日志？
2. 是否能够从日志中看出哪一步失败？

---

## 阶段 4 → 阶段 5：上下文充足性检查

- [ ] 已查阅相关 API Reference
- [ ] 已搜索类似问题（如适用）
- [ ] 已分析项目中类似实现
- [ ] 已确认数据库 Schema（如涉及）
- [ ] 已形成至少 2 个解决方案

**门控问题**:
1. 是否有足够的信息支持修复决策？
2. 是否形成了明确的解决方案？
3. 用户是否已确认采用的方案？

**⚠️ 上下文充足前禁止修复代码**

---

## 阶段 5 → 阶段 6：修复验证检查

- [ ] 已实施代码修复
- [ ] 已重新运行 MCP 调试验证
- [ ] 问题操作不再报错
- [ ] 相关功能正常工作
- [ ] pnpm typecheck 通过
- [ ] pnpm lint 通过
- [ ] pnpm build 通过

**门控问题**:
1. 修复是否已验证成功？
2. 构建是否通过？
