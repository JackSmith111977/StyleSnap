## 1. 文档概述

### 1.1 目的

### 1.2 范围
涵盖 MVP 版本的所有用户可见页面和核心业务流程。

### 1.3 目标读者
- 产品设计师（理解用户流程）
- 前端开发者（实现页面与路由）
- 后端开发者（理解 API 调用场景）
- 测试人员（编写测试用例）

## 4. 核心用户流程

### 4.1 游客浏览风格流程

**触发条件**：用户访问首页或风格列表页

**主成功流**：
```
1. 用户访问 / 或 /styles
2. 系统加载精选轮播（首页）或风格列表
3. 用户浏览风格卡片
4. 用户点击风格卡片
5. 系统跳转至 /styles/:id 风格详情页
6. 用户查看预览、代码、设计变量
7. 用户复制代码片段
8. 流程结束
```

**分支流**：
- 4.1 用户进行搜索 → 跳转 `/search?q=xxx`
- 4.2 用户使用筛选 → 列表实时更新
- 4.3 用户切换视图 → 列表/网格视图切换

---

### 4.2 用户注册流程

**触发条件**：用户点击导航栏「登录/注册」按钮

**主成功流**：
```
1. 用户访问 /auth/register
2. 用户填写邮箱、密码、确认密码
3. 用户点击「注册」按钮
4. 系统验证输入（前端 + 后端）
5. 系统创建用户账户（未验证状态）
6. 系统发送验证邮件
7. 系统显示「请验证邮箱」提示页
8. 用户点击邮件中的验证链接
9. 系统验证令牌，更新用户状态为已验证
10. 系统跳转至首页，显示欢迎 Toast
11. 流程结束
```

**业务异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 邮箱已注册 | 步骤 4 | 邮箱字段下方显示「该邮箱已被注册」 |
| 密码强度不足 | 步骤 4 | 密码字段下方显示强度要求 |
| 密码不匹配 | 步骤 4 | 确认密码字段下方显示「密码不匹配」 |
| 邮箱验证超时 | 步骤 8 | 显示「验证链接已失效，请重新发送」 |

**系统异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 数据库错误 | 步骤 5 | 显示「系统繁忙，请稍后重试」+ 重试按钮 |
| 邮件发送失败 | 步骤 6 | 显示「验证邮件发送失败，请稍后重试」 |
| 网络错误 | 任意步骤 | 显示网络连接错误 + 重试按钮 |

---

### 4.3 用户登录流程

**触发条件**：用户点击导航栏「登录」按钮

**主成功流**：
```
1. 用户访问 /auth/login
2. 用户选择登录方式：
   A. 邮箱 + 密码
   B. GitHub OAuth
   C. Google OAuth
3A. 用户输入邮箱、密码，点击「登录」
4A. 系统验证凭据
5A. 系统创建 Session（30 天有效期）
6A. 系统跳转至首页或登录前页面

3B. 用户点击「GitHub 登录」
4B. 系统跳转至 GitHub OAuth 授权页
5B. 用户授权
6B. 系统处理回调，创建/查找用户
7B. 系统创建 Session，跳转首页

3C. 用户点击「Google 登录」
4C-7C. 同 GitHub 流程
```

**业务异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 邮箱未注册 | 步骤 4A | 显示「该邮箱未注册，请先注册」 |
| 密码错误 | 步骤 4A | 密码字段下方显示「密码错误」 |
| 邮箱未验证 | 步骤 4A | 显示「请先验证邮箱」+ 重新发送按钮 |
| 账户被禁用 | 步骤 4A | 显示「账户已被禁用，请联系管理员」 |
| OAuth 取消授权 | 步骤 5B/5C | 显示「已取消登录」 |
| OAuth 邮箱已存在 | 步骤 6B/6C | 自动绑定到现有账户 |

---

### 4.4 收藏风格流程

**触发条件**：已登录用户在风格详情页点击「收藏」按钮

**前置条件**：用户已登录且邮箱已验证

**主成功流**：
```
1. 用户访问 /styles/:id
2. 用户点击「收藏」按钮
3. 系统切换按钮状态（已收藏/未收藏）
4. 系统发送收藏/取消收藏 API 请求
5. 系统更新本地缓存
6. 显示 Toast 提示「已加入收藏夹」/「已取消收藏」
7. 流程结束
```

**业务异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 未登录 | 步骤 2 | 跳转 /auth/login，登录后返回原页面 |
| 邮箱未验证 | 步骤 2 | 显示「请先验证邮箱」Modal |
| 重复收藏 | 步骤 4 | 幂等处理，返回成功 |

**系统异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 网络错误 | 步骤 4 | 按钮回滚状态，显示「收藏失败，请重试」 |
| 服务器错误 | 步骤 4 | 显示「系统繁忙，请稍后重试」 |

---

### 4.5 发表评论流程

**触发条件**：已登录用户在风格详情页发表评论

**前置条件**：用户已登录且邮箱已验证

**主成功流**：
```
1. 用户访问 /styles/:id
2. 用户滚动至底部评论区域
3. 用户在评论框输入内容
4. 用户点击「发表评论」按钮
5. 系统验证内容（长度、敏感词）
6. 系统发送评论 API 请求
7. 系统将评论添加到评论列表顶部
8. 系统清空评论框
9. 显示 Toast 提示「评论成功」
10. 流程结束
```

**业务异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 评论为空 | 步骤 5 | 评论框下方显示「请输入评论内容」 |
| 评论过长 | 步骤 5 | 显示「评论不能超过 500 字」 |
| 包含敏感词 | 步骤 5 | 显示「评论包含不当内容，请修改」 |
| 重复评论 | 步骤 5 | 显示「请勿重复发表相同内容」 |

**回复评论流程**：
```
1. 用户点击某条评论的「回复」按钮
2. 系统展开回复输入框
3. 用户输入回复内容
4. 用户点击「发送回复」
5-10. 同主成功流
```

---

### 4.6 搜索风格流程

**触发条件**：用户在搜索框输入内容并按 Enter

**主成功流**：
```
1. 用户在任意页面搜索框输入关键词
2. 系统显示自动补全建议（可选）
3. 用户按 Enter 或点击搜索按钮
4. 系统跳转至 /search?q=关键词
5. 系统加载搜索结果列表
6. 用户浏览搜索结果
7. 用户点击搜索结果
8. 系统跳转至风格详情页
9. 流程结束
```

**分支流**：
- 2.1 无补全建议 → 不显示下拉
- 4.1 无搜索结果 → 显示「未找到相关风格」+ 推荐

**业务异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 搜索词为空 | 步骤 3 | 不触发搜索，或显示「请输入搜索内容」 |
| 搜索词过短 | 步骤 3 | 显示「至少输入 2 个字符」 |
| 无搜索结果 | 步骤 5 | 显示空状态 + 推荐风格 |

### 4.7 用户提交风格流程

**触发条件**：已登录用户点击「提交风格」按钮

**前置条件**：用户已登录且邮箱已验证

**主成功流**：
```
1. 用户访问 /user/submissions 或点击「提交风格」
2. 系统显示提交流程引导
3. 用户填写基本信息（名称、描述、分类、标签）
4. 用户上传预览图（支持拖拽上传）
5. 用户填写代码片段（HTML/CSS、React、Tailwind）
6. 用户预览提交内容
7. 用户点击「提交审核」
8. 系统验证数据完整性
9. 系统创建提交记录（状态：pending）
10. 系统跳转至「我的提交」页
11. 显示 Toast 提示「提交成功，等待审核」
12. 流程结束
```

**审核流程（后台）**：
```
1. 管理员访问 /admin/submissions
2. 系统加载待审核列表
3. 管理员点击某条提交
4. 系统显示提交详情
5. 管理员选择「通过」或「拒绝」
6. 如拒绝，管理员填写拒绝原因
7. 管理员确认操作
8. 系统更新提交状态（approved/rejected）
9. 系统发送站内通知给用户
10. 如通过，系统将风格发布到公开列表
11. 流程结束
```

**业务异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 必填项为空 | 步骤 8 | 高亮空字段，显示「请填写完整信息」 |
| 图片格式不支持 | 步骤 4 | 显示「仅支持 PNG、JPG 格式」 |
| 图片过大 | 步骤 4 | 显示「图片不能超过 5MB」 |
| 代码为空 | 步骤 8 | 显示「请至少填写一种代码片段」 |

---

### 4.8 忘记密码流程

**触发条件**：用户在登录页点击「忘记密码」

**主成功流**：
```
1. 用户访问 /auth/forgot
2. 用户输入注册邮箱
3. 用户点击「发送重置邮件」
4. 系统验证邮箱是否存在
5. 系统发送密码重置邮件（含重置链接）
6. 系统显示「重置邮件已发送」提示页
7. 用户点击邮件中的重置链接
8. 系统验证令牌有效性
9. 用户输入新密码并确认
10. 用户点击「重置密码」
11. 系统更新密码，使旧 Session 失效
12. 系统显示「密码重置成功」
13. 系统跳转至登录页
14. 流程结束
```

**业务异常流**：
| 异常 | 触发条件 | 系统处理 |
|------|----------|----------|
| 邮箱未注册 | 步骤 4 | 显示「该邮箱未注册」（或模糊提示防止探测） |
| 令牌失效 | 步骤 8 | 显示「重置链接已失效，请重新申请」 |
| 密码强度不足 | 步骤 10 | 显示密码强度要求 |
| 密码不匹配 | 步骤 10 | 显示「两次输入的密码不一致」 |

## 6. 异常流处理

### 6.1 认证异常

| 异常类型 | HTTP 状态 | 用户可见提示 | 系统处理 |
|----------|-----------|--------------|----------|
| 未登录访问受限页 | 401 | 「请先登录后再访问」 | 跳转登录页，携带 redirect 参数 |
| 无权访问 | 403 | 「您没有权限访问此页面」 | 显示 403 页，返回首页 |
| Token 过期 | 401 | 「登录已过期，请重新登录」 | 清除 Session，跳转登录 |
| 邮箱未验证 | 403 | 「请先验证邮箱」 | 显示验证 Modal |

### 6.2 网络异常

| 场景 | 用户可见提示 | UI 表现 |
|------|--------------|----------|
| 列表加载失败 | 「加载失败，点击重试」 | 显示错误状态 + 重试按钮 |
| 图片加载失败 | 占位图（带错误图标） | 替换为错误占位图 |
| 表单提交失败 | 「提交失败，请检查网络连接」 | 表单顶部显示错误 + 重试按钮 |
| 无限滚动加载失败 | 「加载更多失败，点击重试」 | 列表底部显示错误 + 重试按钮 |

### 6.3 业务异常

| 场景 | 用户可见提示 | UI 表现 |
|------|--------------|----------|
| 重复提交 | 「请勿重复操作」 | Toast 提示 |
| 数据验证失败 | 具体字段错误信息 | 字段下方红色文字 |
| 资源不存在 | 「该内容不存在或已被删除」 | 404 页 |
| 操作频率限制 | 「操作过于频繁，请稍后再试」 | Toast 提示 |

## 7. 导航与布局

### 7.1 顶部导航栏

**布局结构**：
```
┌────────────────────────────────────────────────────────────┐
│ [Logo]  [风格] [搜索]              [主题切换] [用户头像 ▼] │
│          （移动端：汉堡菜单）                               │
└────────────────────────────────────────────────────────────┘
```

**状态**：
- 固定顶部（`position: sticky`）
- 滚动时显示/隐藏逻辑（可选）
- 移动端：汉堡菜单 + 抽屉式侧边栏

### 7.2 面包屑导航

**显示位置**：页面标题上方

**示例**：
```
首页 > 风格列表 > 科技未来 > 详情页
```

**显示规则**：
- 首页不显示面包屑
- 风格详情页显示完整路径
- 管理后台显示 `/ 管理后台 / 当前页`

### 7.3 页脚

**内容**：
- Logo + 产品简介
- 快速链接（关于、隐私政策、服务条款）
- 社交媒体链接
- 版权信息

### 7.4 主题切换

**位置**：导航栏右上角

**行为**：
- 点击切换深色/浅色模式
- 默认跟随系统
- 记住用户偏好（LocalStorage）
